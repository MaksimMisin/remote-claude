## Codebase Patterns
- `escapeHtml()` in `telegram-format.ts` for all user-facing text in Telegram HTML
- `markdownToTelegramHtml()` for converting Claude's markdown responses to Telegram HTML
- `splitMessage()` in `telegram-format.ts` handles 4000 char Telegram message limit
- Pre-existing typecheck errors in `server/TopicManager.test.ts` — ignore when running `npx tsc --noEmit`
- Event buffer in TelegramBot.ts: `eventBuffer` Map stores per-session `DigestEvent[]` arrays, flushed every 10s via `formatActivityDigest()`
- `formatEventLine()` returns per-tool emoji + description lines (still used for standalone tool display, not digest)
- `onEvent()` routes: pre_tool_use → digest buffer, ExitPlanMode → standalone plan, stop → standalone, user_prompt_submit → initialPrompt tracking only
- `sendMessage()` in TelegramBot.ts already calls `splitMessage()` — no need to manually split in callers
- `formatSessionFinished()` three tiers: <150 chars = inline "finished — summary", 150-500 = header + text, 500+ = expandable blockquote. Prefers markerMsg over snippet.
- `session.planContent` stores plan from ExitPlanMode — captured on pre_tool_use, cleared on user_prompt_submit
- Variables for delayed setTimeout idle notifications must be captured before the delay (closure pattern: `capturedPlanContent`, `capturedMarkerCategory`)
